# User Input Contract for Game Generation

This is the product-facing input contract for Level 1 game generation.

The user should not need to know numeric game parameters such as `spawnInterval`, `gravity`, or `jumpVelocity`. Those remain model/recipe responsibilities.

## Minimal Input

The user can enter one sentence:

```text
做一个轻松的太空跑酷小游戏，适合小朋友，画面可爱一点。
```

The system should infer:

- `genre`: default `runner` unless clearly specified
- `theme`: `space`
- `difficulty`: `easy`
- `audience`: kids/casual
- `mood`: playful
- `reward`: survival or score
- safe numeric recipe

## Recommended UI Fields

For a better product experience, collect these fields explicitly:

1. **Game idea** — free text
   - Example: `做一个赛博朋克跑酷小游戏`
2. **Audience** — optional select
   - `kids`, `casual`, `core`, `event-page`
3. **Difficulty** — select
   - `easy`, `normal`, `hard`
4. **Mood** — optional select
   - `calm`, `playful`, `intense`, `mysterious`
5. **Session length** — optional select
   - `short`, `medium`, `endless`
6. **Visual theme** — optional free text/select
   - Examples: `space`, `candy`, `cyberpunk`, `forest`, `ocean`
7. **Iteration instruction** — used after first preview
   - Examples: `更简单一点`, `换成太空主题`, `节奏快一点`, `更适合小朋友`

## What The User Should Not Input

Do not expose these unless in an advanced/debug mode:

- `spawnInterval`
- `scrollSpeed`
- `jumpVelocity`
- `gravity`
- obstacle pixel width/height
- robot/eval thresholds

These should be generated by the model and then clamped by recipe/validator.

## Model Output Responsibilities

The model should output `config.json` with:

- required gameplay fields from `docs/gdd-schema.md`
- runner playability recipe preserved
- optional `experience` fields:
  - `goalText`
  - `instructionText`
  - `successText`
  - `failureText`
  - `mood`
  - `reward`

## Product Loop

```text
User idea
  -> intent extraction
  -> config generation
  -> schema + recipe validation
  -> robot eval if needed
  -> playable preview
  -> user iteration instruction
  -> config revision
```

## Example Inputs

```text
做一个轻松的糖果跑酷，适合 6 岁小朋友，失败提示要鼓励一点。
```

```text
做一个赛博朋克跑酷，难一点，有追逐感，但不要一开始就死。
```

```text
做一个活动页小游戏，30 秒内能玩完，主题是海底寻宝。
```

```text
把当前游戏改得更治愈一点，速度慢一点，提示更友好。
```
